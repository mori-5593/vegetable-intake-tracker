## サービス概要
1日の野菜摂取量を直感的に可視化できるアプリです。
野菜ごとに摂取量を記録すると、イラストの変化によって摂取状況がわかります。
合計量が基準値を超えると、その日の目標達成を確認できます。
＊正確な栄養管理ではなく、日々の意識づけを目的としています。

## このサービスへの思い・作りたい理由
大人になり日々の食事管理を自分でおこなっていく中で、気づけば偏った食事ばかりになっていると感じました。
健康な食生活を送りたい、でも何を食べれば良いか分からない。
栄養バランスの整った献立を考えるのが面倒。

近年では、コンビニエンスストアでも「1日の野菜摂取1/2量分！」と書かれたお弁当などをよく見かけます。
これは自炊をしていく中でも自分で出来ることなのではないかと思い、正確な栄養管理ではなく野菜の摂取量を目安として可視化することで、日々の意識づけを行えるアプリを作成していきたいと考えました。

このサービスを通して自分の健康管理の始めの取っ掛かりとして手に取ってもらえればと思います。

## ユーザー層について
・健康を意識し始めた人
・食生活を見直したいと感じている人
栄養管理アプリの多くは、数値入力や専門知識を前提としており、継続のハードルが高いと感じました。
そこで本サービスでは、まずは野菜の摂取量を直感的に可視化することに焦点を当て、気軽に使えるユーザー層を対象としています。

## サービスの利用イメージ
| 課題・問題点 | サービスで行うこと | 得られる価値 |
|--------------|------------------|--------------|
| 1日にどの野菜をどれだけ食べているか分からない | 摂取した野菜をタップ操作で記録する | その日の野菜摂取量を可視化できる |
| 数値入力や栄養計算が面倒で継続できない | イラストの変化によって摂取量を表現する | 直感的に状況を把握でき、入力の負担が減る |
| 野菜を摂取できているか判断基準がない | 合計摂取量を基準値と比較する | 目標を達成できているかが一目で分かる |
| 健康を意識しても行動が続かない | 達成時に視覚的な達成表示を行う| 日々の野菜摂取を意識するきっかけになる |

## ユーザーの獲得について
本サービスは、個人開発として作成を予定しており、大規模なユーザー獲得は想定しておりません。
まずは、日々の食生活を見直したいと感じている自分自身や、同様の悩みを持つ身近な人に利用してもらうことを想定しています。

## サービスの差別化ポイント・推しポイント
正確な栄養計算や詳細な入力を行わず、野菜の摂取量を直感的に可視化する点にあります。
一般的な栄養管理アプリでは、食材の細かな検索や数値入力が必要となり、継続のハードルが高いと感じることがありました。
本サービスでは、野菜ごとにイラストを用いたタップ操作で摂取量を記録できるため、操作の負担を抑えつつ、日々の摂取状況を視覚的に把握することができます。

## 機能候補
MVPリリース時に作っていたいもの
・野菜一覧の表示機能
・野菜ごとの摂取量をタップ操作で記録する機能
・摂取量に応じてイラストが変化する表示機能
・1日の合計摂取量を算出し、基準値と比較する機能
・目標達成/未達成を表示する機能

本リリースまでに作っていたいもの
・野菜の種類を追加、管理できる機能
・連続達成日数などの簡易的な達成履歴表示
・摂取量のリセットや修正を行う機能

## 使用する技術スタック
- 使用するフレームワーク：Ruby on Rails
- データベース：PostgreSQL
- デプロイ先：Render
- 使用予定のライブラリ：Hotwire（Turbo / Stimulus）
タップ操作による摂取量の更新や、イラストの表示切り替えをページ遷移なしで行うことを想定しています。
野菜ごとに摂取段階（例：0〜3）を状態として保持し、その値に応じて表示するイラストを切り替えることで、摂取量を直感的に可視化する実装を想定しています。

## 認証について
本サービスは、日々の野菜摂取量を手軽に記録することを目的としているため、現時点ではユーザー登録、ログイン機能は設けない想定としています。
食事のたびに素早く入力できることを重視し、初回からすぐ利用できる体験を優先しました。
